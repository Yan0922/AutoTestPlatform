# 算法测试平台 — 完整部署教程（已实测通过）

> 本教程**手把手**带你从零部署到可以在浏览器使用。
> 后端：Django + DRF；前端：Vue 3 + Element Plus；数据库：**MySQL 8**。
> 所有步骤已在 Windows 10 / Python 3.13 / Node 22 上**实际验证通过**。

---

## 一、需要先准备好的工具

| 工具 | 推荐版本 | 必需性 | 备注 |
|---|---|---|---|
| Python | 3.10 ~ 3.13 | ✅ 必需 | 你机器已经有 3.13，OK |
| Node.js + npm | 18+ | ✅ 必需 | 你机器已经有 22，OK |
| MySQL Server | 8.0 | ✅ 必需 | **下面教你装** |
| Git | 任意 | 可选 | 仅用于版本控制 |

> 不需要单独装 `mysqlclient`、`Visual Studio C++` 这些编译工具，本项目用纯 Python 的 **PyMySQL** 当驱动，**Windows 上零编译**。

## 二、安装 MySQL（Windows 篇）

### 方式 A：MySQL 官方 Installer（推荐）

1. 进入 [MySQL 官方下载页](https://dev.mysql.com/downloads/installer/) 下载 **MySQL Installer for Windows** 离线包（约 400MB 那个，叫 `mysql-installer-community-x.x.x.msi`）。

2. 打开安装包，选择 **Custom**，只勾选两个：
   - `MySQL Server 8.0.x`
   - `MySQL Workbench`（可视化客户端，非必需但强烈建议装）
   点 Next → Execute 一路装完。

3. 进入 Configuration 页：
   - **Type and Networking**：选 *Development Computer*，Port 保持 `3306`
   - **Authentication Method**：选 *Use Strong Password Encryption*
   - **Accounts and Roles**：设置 root 密码（**记住这个密码！** 例如 `Root@123`）
   - **Windows Service**：勾选 *Start the MySQL Server at System Startup*（开机自启）
   - 一路 Next → Finish

4. **验证安装**：把 MySQL bin 目录加进 PATH（默认是 `C:\Program Files\MySQL\MySQL Server 8.0\bin`），然后新开一个 PowerShell：

   ```powershell
   mysql --version
   # 应该输出: mysql  Ver 8.0.x ...
   ```

### 方式 B：用 Docker（如果你已经有 Docker Desktop）

```powershell
docker run -d --name algotest-mysql `
  -p 3306:3306 `
  -e MYSQL_ROOT_PASSWORD=Root@123 `
  -e MYSQL_DATABASE=algotest `
  -e MYSQL_USER=algotest `
  -e MYSQL_PASSWORD=algotest123 `
  --restart unless-stopped `
  mysql:8.0 --character-set-server=utf8mb4 --collation-server=utf8mb4_unicode_ci
```

如果走 Docker，**第三步可以跳过**（库和账号已经在这条命令里建好了）。

## 三、在 MySQL 里创建数据库和账号

打开 MySQL Workbench（或命令行 `mysql -u root -p`）连进去 root 账号，执行：

```sql
CREATE DATABASE algotest CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
CREATE USER 'algotest'@'%' IDENTIFIED BY 'algotest123';
GRANT ALL PRIVILEGES ON algotest.* TO 'algotest'@'%';
FLUSH PRIVILEGES;
```

> 用户名 / 密码可以改成你自己想要的，记得**和下面 `.env` 文件保持一致**。

## 四、启动后端（Django）

### 4.1 配置环境变量

进入项目后端目录，把 `.env.example` 复制成 `.env`：

```powershell
cd D:\Projects\AutoTestPlatform\backend
Copy-Item .env.example .env
```

打开 `.env`，确认这几行对应到你第三步设置的数据库账号密码：

```
DB_ENGINE=mysql
DB_NAME=algotest
DB_USER=algotest
DB_PASSWORD=algotest123
DB_HOST=127.0.0.1
DB_PORT=3306
```

> ⚠️ Windows 上的 Django 不会自动加载 `.env`，请用下面的 PowerShell 方式注入：

```powershell
# 临时注入环境变量（每个新的 PowerShell 窗口要重做一次）
Get-Content .env | ForEach-Object {
    if ($_ -and $_ -notmatch '^\s*#') {
        $kv = $_ -split '=', 2
        [Environment]::SetEnvironmentVariable($kv[0].Trim(), $kv[1].Trim(), 'Process')
    }
}
```

或者更简单：直接在当前 PowerShell 设置：

```powershell
$env:DB_ENGINE   = 'mysql'
$env:DB_NAME     = 'algotest'
$env:DB_USER     = 'algotest'
$env:DB_PASSWORD = 'algotest123'
$env:DB_HOST     = '127.0.0.1'
$env:DB_PORT     = '3306'
```

### 4.2 创建虚拟环境、安装依赖

```powershell
cd D:\Projects\AutoTestPlatform\backend
python -m venv venv
.\venv\Scripts\Activate.ps1    # 如果提示无法执行脚本，先运行 Set-ExecutionPolicy -Scope CurrentUser RemoteSigned
pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple
```

> **第一次 pip install 会下载约 60MB 的包**，耐心等 1~2 分钟。

### 4.3 初始化数据库（核心一步）

**所有表都是 Django 通过 `migrate` 自动建的，不用手写 SQL**。

```powershell
python manage.py migrate
```

成功输出类似：

```
Operations to perform:
  Apply all migrations: admin, asr, auth, contenttypes, sessions
Running migrations:
  Applying contenttypes.0001_initial... OK
  Applying asr.0001_initial... OK
  ...
```

执行后到 MySQL 里 `SHOW TABLES;` 就能看到我们的 8 张业务表：

```
asr_model
asr_model_file
audio
dataset
dataset_audio
test_task
test_task_dataset
test_audio_result
```

### 4.4 创建 admin 用户（可选）

```powershell
python manage.py createsuperuser
# 按提示输入用户名 / 邮箱 / 密码
```

之后可以用 `http://127.0.0.1:8000/admin/` 登录后台直接管理数据。

### 4.5 灌入演示数据（强烈推荐第一次）

```powershell
python manage.py seed_demo
```

会生成 2 个 ASR 模型、2 个数据集、20 条音频，方便看到效果。

### 4.6 启动后端服务

```powershell
python manage.py runserver 0.0.0.0:8000
```

看到以下输出就是启动成功：

```
Starting development server at http://127.0.0.1:8000/
```

**保持这个窗口开着不要关**，浏览器打开 http://127.0.0.1:8000/api/asr/models/ 应该能看到 JSON 数据。

## 六、跑一次完整的体验流程（验证一切正常）

1. 进入 **ASR → 模型管理**：能看到演示模型列表（如果跑过 `seed_demo`）。
2. 点 **上传模型**，填写名称/版本，选一个本地小文件夹上传（任何文件夹都行，比如随便建个文件夹放几个 txt）。点击「提交上传」，列表会刷新出新的一条。
3. 进入 **ASR → 数据集管理**，能看到两个演示数据集，点击右侧能看到关联的音频。
4. 进入 **ASR → 数据池**：能看到 20 条音频，支持多条件筛选 / 文本编辑 / Excel 导入。
5. 进入 **ASR → 测试任务管理**，点 **创建任务**：
   - 任务名称：随便填
   - 模型：选演示模型
   - 数据集：勾选两个数据集
   - 提交，立刻就能看到任务状态从「进行中」变成「运行完成」（我用的是模拟推理，速度很快）。
6. 点 **运行结果** → 新页签打开结果详情：
   - 左侧两个数据集卡片，含 总音频数 / 总时长 / WER / S / I / D / Hit 等指标
   - 右侧是每条音频的对比，**红色标记**就是 ASR 识别错误的字符
   - 点 Ref 文本可以编辑，OK 后会同步回数据池

## 七、常见问题（FAQ）

### Q1：runserver 启动报错 `(2003, "Can't connect to MySQL server")`

→ MySQL 服务没启动。Windows 上：
- 按 `Win + R` 输入 `services.msc`
- 找到 *MySQL80* 这一项，右键 → **启动**

### Q2：`(1045, "Access denied for user 'algotest'@'localhost'")`

→ 数据库账号密码不对。重新执行第三步的 SQL，或者把 `.env` 改成 root 账号试一下。

### Q3：`ModuleNotFoundError: No module named 'pymysql'`

→ 没在虚拟环境里。请先 `.\venv\Scripts\Activate.ps1` 再 `pip install -r requirements.txt`。

### Q4：前端报 `ECONNREFUSED 127.0.0.1:8000` / 接口都是红色

→ 后端没起，或者后端在其他端口。前端通过 `/api` 自动代理到 `127.0.0.1:8000`，把后端起在 8000 就行。

### Q5：上传文件夹时浏览器弹窗只能选单个文件

→ 一定要用 Chrome / Edge 浏览器，并且选目录而不是文件。代码里用了 `webkitdirectory` 属性，IE / 老版本浏览器不支持。

### Q6：我不想装 MySQL，能不能临时用 SQLite

→ 可以。把 `.env` 里的 `DB_ENGINE=mysql` 改成 `DB_ENGINE=sqlite`，再 `python manage.py migrate` 就行。**生产环境建议还是 MySQL**。
